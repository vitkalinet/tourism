# 📝 Как добавлять и редактировать экскурсии

```
tourism/
├── data/
│   └── tours/                  ← БАЗА ДАННЫХ ЭКСКУРСИЙ (каждая экскурсия в отдельном файле)
│       ├── categories.json     ← Категории экскурсий
│       ├── khurul.json         ← Экскурсия «Буддизм легко и доступно»
│       ├── elista-walk.json    ← Экскурсия «Прогулка по Элисте»
│       ├── topol.json          ← Экскурсия «Одинокий тополь и родники»
│       ├── palmov-museum.json  ← Экскурсия «Музей Пальмова»
│       ├── dunes-saigas.json   ← Экскурсия «Поющие барханы и сайгаки»
│       ├── nomads-camels.json  ← Экскурсия «Кочевники и верблюды»
│       ├── horse-riding.json   ← Экскурсия «Конные прогулки»
│       ├── lotus.json          ← Экскурсия «Цветение лотосов»
│       ├── tibetan-medicine.json
│       ├── massage.json
│       ├── yurt-night.json
│       └── chess-bender.json

│   ├── app/
│   │   ├── tour/

└── public/                     ← Статические файлы
```

## ✏️ Редактирование экскурсии

### 1. Откройте нужный файл в `data/tours/`

Каждая экскурсия — отдельный JSON-файл. Например:
- `data/tours/khurul.json` — экскурсия «Буддизм легко и доступно»
- `data/tours/lotus.json` — экскурсия «Цветение лотосов»

### 2. Структура файла экскурсии

```json
{
  "id": "unique-id",           ← Уникальный идентификатор (латиницей, без пробелов)
  "slug": "unique-id",         ← URL-адрес: /tour/unique-id
  "title": "Название",         ← Заголовок на странице
  "subtitle": "Подзаголовок",  ← Краткое описание в карточке
  "description": "...",        ← Короткое описание (для каталога)
  "shortDescription": "...",   ← Продающее описание (3-4 предложения)
  "fullDescription": "...",    ← Полное описание экскурсии
  "duration": "2 часа",        ← Длительность
  "price": "от 1500 ₽",        ← Цена
  "category": "culture",       ← Категория (см. ниже)
  "categoryLabel": "Культура", ← Отображаемое название категории
  "icon": "☸",                 ← Эмодзи-иконка
  "highlights": [...],         ← Ключевые точки маршрута
  "gradient": "...",           ← Градиент для фона hero-блока
  "season": "Круглогодично",   ← Сезон
  "image": "🏛",               ← Эмодзи для карточки
  "itinerary": [...],          ← Программа по времени
  "included": [...],           ← Что включено
  "notIncluded": [...],        ← Что не включено
  "importantInfo": [...],      ← Важная информация
  "related": [...]             ← Связанные экскурсии (по slug)
}
```

## ➕ Добавление новой экскурсии

1. **Создайте файл** `data/tours/new-tour.json` (имя файла = slug)
2. **Скопируйте структуру** из любого существующего файла
3. **Заполните все поля**:

```json
{
  "id": "new-tour",
  "slug": "new-tour",
  "title": "Новая экскурсия",
  "subtitle": "Краткое описание",
  "description": "Описание для каталога",
  "shortDescription": "Продающее описание",
  "fullDescription": "Полное описание",
  "duration": "3 часа",
  "price": "от 2000 ₽",
  "category": "nature",
  "categoryLabel": "Природа",
  "icon": "🌄",
  "highlights": ["Точка 1", "Точка 2"],
  "gradient": "linear-gradient(135deg, #color1, #color2)",
  "season": "Май — Сентябрь",
  "image": "🌄",
  "itinerary": [
    { "time": "10:00", "title": "Встреча", "desc": "Описание" }
  ],
  "included": ["Трансфер", "Гид"],
  "notIncluded": ["Питание"],
  "importantInfo": [
    { "label": "Длительность", "value": "3 часа" }
  ],
  "related": ["khurul", "topol"]
}
```

4. **Добавьте импорт в `src/data/tours.ts`**:

```typescript
import newTour from "../../data/tours/new-tour.json";

// и добавьте в массив:
export const tours: Tour[] = [
  // ...существующие...
  newTour,
];
```

5. Перезапустите dev-сервер: `npm run dev`

## 📝 Редактирование программы маршрута

```json
"itinerary": [
  {
    "time": "10:00",
    "title": "Название этапа",
    "desc": "Подробное описание"
  },
  {
    "time": "11:00",
    "title": "Следующий этап",
    "desc": "Описание"
  }
]
```

Добавляйте сколько угодно этапов — просто копируйте объект.

## 🏷 Изменение категорий

Категории хранятся в `data/tours/categories.json`:

```json
[
  { "id": "all", "label": "Все экскурсии" },
  { "id": "culture", "label": "Культура" },
  { "id": "nature", "label": "Природа" },
  { "id": "adventure", "label": "Приключения" },
  { "id": "wellness", "label": "Оздоровление" }
]
```

Чтобы добавить новую категорию:
1. Добавьте объект в `categories.json`
2. Укажите `category` и `categoryLabel` в файле экскурсии

## 🔗 Связанные экскурсии

Поле `related` содержит slug-ы других экскурсий. Они отображаются в блоке "Вам также может понравиться".

## 🎨 Настройка цветов

### Градиенты для hero-блока

Используйте CSS-градиенты:
```
linear-gradient(135deg, #color1 0%, #color2 100%)
```

Готовые варианты:
- 🟡 Золотой: `linear-gradient(135deg, #c9a84c 0%, #e8c97a 100%)`
- 🟢 Зелёный: `linear-gradient(135deg, #43e97b 0%, #38f9d7 100%)`
- 🔵 Синий: `linear-gradient(135deg, #2d7a7a 0%, #4da8a8 100%)`
- 🟠 Терракотовый: `linear-gradient(135deg, #b8651f 0%, #d4a574 100%)`
- 🌸 Розовый: `linear-gradient(135deg, #fa709a 0%, #fee140 100%)`

## 📱 Изменение контактов

Max номер указан в компонентах. Найдите и замените:
```
79001234567  ← Ваш номер Max
```

## ❗ Важные правила

1. **Имя файла** = `slug` экскурсии (например, `khurul.json`)
2. **id** и **slug** должны быть уникальными и совпадать
3. **related** — только slug-ы существующих экскурсий
4. **category** — только из списка в `categories.json`
5. **itinerary** — массив объектов с полями `time`, `title`, `desc`
6. **included**, **notIncluded**, **highlights** — массивы строк
7. **importantInfo** — массив объектов с `label` и `value`
8. При добавлении новой экскурсии — не забудьте обновить `src/data/tours.ts`
